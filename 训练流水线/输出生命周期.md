# `输出/` 生命周期管理说明

## 1. 目标

这套机制的目的不是训练模型，而是治理 `输出/` 目录，避免运行时间久了以后：

- 历史产物无限堆积
- 关键结果被新一轮训练覆盖
- 备份目录占用过多空间

当前策略重点是：

- 自动快照关键产物
- 自动清理超量历史目录
- 输出清理报告

## 2. 当前接入位置

- 主入口：`训练流水线/运行训练.py`
- 实现模块：`训练流水线/数据流水线/输出生命周期.py`
- 输出报告：`输出/输出生命周期报告.json`

此外也可以独立执行：

```bash
python -m 训练流水线.管理输出 --config 训练流水线/配置/高精度.yaml
```

## 3. 当前配置项

配置文件：`训练流水线/配置/高精度.yaml`

当前关注的配置块如下：

```yaml
outputs_lifecycle:
  enabled: true
  dry_run: false
  archive_dir: "输出/归档运行"
  backup_dir: "输出/备份"
  keep_latest_runs: 12
  keep_latest_backups: 8
  max_backup_age_days: 30
  require_age_for_backup_delete: false
  snapshot_files:
    - "模型训练报告.json"
    - "回测报告.json"
    - "推荐结果.csv"
    - "高精度训练报告.md"
```

## 4. 实际执行逻辑

根据 `训练流水线/数据流水线/输出生命周期.py`，当前逻辑是：

1. 先把 `snapshot_files` 复制到 `archive_dir/<run_tag>/`
2. 只保留最近 `keep_latest_runs` 个归档目录
3. 清理 `backup_dir` 下多余的备份目录
4. 如启用 `require_age_for_backup_delete=true`，还会要求备份年龄超过 `max_backup_age_days`
5. 把删除结果、释放空间和失败项写入报告

## 5. 关键实现细节

### 快照阶段

- 即使某些文件不存在，也不会直接失败
- 不存在的文件会被记入 `missing_files`

### 删除阶段

- 对归档目录按修改时间从新到旧排序
- 超出保留数量的目录会被删除
- 支持 `dry_run`

### 备份目录清理

- 默认只按“保留最近 N 个”处理
- 如果启用年龄限制，再叠加“备份足够旧”这一条件

## 6. 这套机制解决了什么问题

对当前项目来说，它主要解决三类问题：

- `输出/` 下训练产物太多，不易定位最近一次稳定结果
- 回测、推荐和训练报告容易被新一轮运行覆盖
- `输出/备份/` 会随着兼容重训练持续膨胀

## 7. 当前使用建议

- 第一次调整策略时，先把 `dry_run` 设为 `true`
- 如果担心清理过度，可以增大：
  - `keep_latest_runs`
  - `keep_latest_backups`
  - `max_backup_age_days`
- 如果机器磁盘空间紧张，再逐步收紧策略

## 8. 与当前项目现状的关系

当前仓库里的 `输出/` 目录已经包含：

- 大量模型文件
- 多次备份目录
- 推荐结果
- 训练报告
- 诊断报告
- 校准器产物

所以这套生命周期管理不是可有可无的附加功能，而是防止 `输出/` 继续失控增长的必要治理手段。
